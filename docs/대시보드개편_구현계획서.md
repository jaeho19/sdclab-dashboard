### 2-1. 졸업 로드맵 (`/roadmap`)
1. **오늘의 브리핑**: D-day 칩 가로 스트립. 칩 = D-day(14일 이내 red·45일 이내 amber) + 제목 + 대상 + 설명 한 줄. 확정일 없는 마감은 "D-?" 점선 칩. D-day는 **클라이언트 JS로 계산**(빌드 시점 아님 — 재빌드가 며칠 멈춰도 정확해야 함).
2. **학생별 여정**: "박사과정 · N명" → "석사과정 · N명" 그룹 순. 카드 구성:
   - 이름 + 과정 배지 + 졸업 목표 학기 배지
   - 연구 키워드 · 진로 1순위
   - `지금`(오늘 날짜와 겹치는 일정, **학회 항목 제외**) · `다음`(다가오는 심사·투고만 — 학회 제외) + D-day
   - 게재 논문이 있으면 `게재 ✓ 제목 — 저널 (상태)` 실명 표기
   - **학기 분절 바**: 지난 학기 = 진한 채움, 남은 학기 = 연회색 채움+테두리(시인성), 우측 "N학기 지남 · **N학기 남음**(굵게)"
   - 요건은 색상 없이 텍스트만: "7개 중 N 완료 · N 진행 · 미착수 N"
3. **졸업요건 히트맵**: 6행(학생) × 7열(이수학점·어학·종합시험·논문자격·예비심사·본심사·최종제출). 완료=초록 ✓ / 진행=호박 ~ / 미착수=점선 테두리. 셀 hover 시 규정 기준+현재 상태 툴팁.
4. ~~확인 필요 신호~~ — **넣지 않는다** (사용자가 제거 지시).

### 2-2. 학생 상세 (`/students/[slug]`)
순서대로: ① 헤더(이름·과정·키워드·입학/목표/진로/진로 요구 산출물/현재 격차 메타 + 학기 바) ② **7노드 졸업 스테퍼**(지하철 노선도형, 완료=초록 ✓·진행=주황 펄스 테두리·미착수=회색, 노드 아래 현재 상태·예정 시기, 범례 필수) ③ **졸업요건 상세** 테이블(상태 배지·항목·기준·현재) ④ **투고 논문**(그 학생의 실제+계획 파이프라인, 상태 배지+날짜) ⑤ **학회 발표 계획**(학회명·발표 제목 가안·준비 기간 연월·D-day 배지, 60일 이내 amber, 제목 미정이면 "시트에 기입하면 표시됩니다") ⑥ 연구 정체성(방법론×데이터 태그 + RQ, 미작성이면 작성 유도 empty state) + 이번 분기/다가오는 일정 2열 ⑦ **개인 간트**(분기 눈금, 연구=파랑·학회=주황·투고=초록·심사=분홍 막대, 오늘선, 시트 '5. 일정' 그대로) ⑧ 지도교수 코멘트 말풍선 스레드(교수=amber 좌측 보더 / 학생 답변 우측, 미응답이면 '응답 대기' 배지).

### 2-3. 투고 논문 (`/papers`)
- **실제 투고된 논문만** 표시. 구상·집필 단계 계획은 이 탭에 넣지 않는다(학생 상세에서만).
- KPI 타일 5개(투고 중 총계·심사 중·수정/재투고·거절·아카이브) → 트래커에서 자동 집계.
- **5단 칸반**: 투고 완료 → 심사 중 → 수정 중 → 재투고 → 거절. 카드 = 제목·KCI/SCI급 배지·원문 상태 영문 배지(Submitted 파랑/Under Review 노랑/Under Revision 주황/Resubmitted 파랑/Rejected 빨강)·주저자·저널·투고 연월(없으면 "기입 필요").
- **아카이브 섹션**(하단): 게재 완료·Accepted 행 리스트 — 상태 배지·제목·등급·저널·주저자·게재 연월.
- 필터 칩: 주저자(데이터에서 동적 생성) · KCI/SCI급 · 연도. **날짜는 연월(YYYY.MM) 단위**로만 다룬다.
- 학회 레이더: 타깃 학회 카드(개최 시기·초록 마감·발표 예정자 아바타 스택·발표 제목). 마감 미확정은 D-day 대신 물음표 배지.

### 2-5. 웹 편집(쓰기) 기능 — 필수 요구 (2026-07-20 추가)

목업의 투고 논문 탭에 **동작하는 데모**가 들어 있다(드래그·연월 입력이 화면에서 실제로 작동 — 로컬만 반영). 실제 구현은 같은 UX로 시트에 기록한다.

1. **칸반 드래그 앤 드롭**: 카드를 다른 컬럼에 놓으면 ① 상태 즉시 변경(낙관적 반영) ② 시트 `06_투고논문`의 상태 컬럼 + **상태변경일 = 오늘 날짜 자동 기록** ③ 성공 토스트 / 실패 시 원위치 롤백 + 오류 토스트. 영문 상태 매핑: 투고 완료=Submitted · 심사 중=Under Review · 수정 중=Under Revision · 재투고=Resubmitted · 거절=Rejected. 아카이브 섹션도 드롭 대상(게재 처리 = 상태 '게재확정').
2. **연월 인라인 편집**: 카드·아카이브 행의 투고/게재 연월(✎ 표시)을 클릭 → `<input type="month">` → 확정 즉시 시트 기록 + 화면 반영.
3. **기타 인라인 편집(같은 메커니즘)**: 논문 제목·저널명 클릭 편집(잘린 제목 복구용). 졸업 로드맵 쪽 편집(요건 상태 토글 등)은 학생 탭 셀 좌표 매핑이 필요해 **2차**로 미룬다 — 1차에서는 각 요소에 '시트에서 열기' 딥링크(gid 앵커)로 대신한다.
4. UX 규칙: 모든 쓰기는 낙관적 반영 → 실패 시 롤백. 저장 중 표시(카드 살짝 흐림). 새로고침해도 최신이 보여야 하므로 투고 논문 탭 데이터는 클라이언트 라이브 로드(§4).
