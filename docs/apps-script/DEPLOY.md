# Phase 3 백엔드 배포 가이드 (사용자 액션)

투고 논문 탭의 **드래그로 상태 변경 · 연월/제목 인라인 편집**을 켜려면 아래 2가지가 필요합니다.
둘 다 **시트 소유 구글 계정**에서 한 번만 하면 됩니다. 끝나면 **웹 앱 URL + 토큰**을 저에게 알려주세요 — 클라이언트에 연결하고 왕복 테스트하겠습니다.

---

## ① 트래커를 구글 시트로 가져오기

지금 「학생 졸업계획서」 시트엔 `06_투고논문` 탭이 없습니다(대시보드는 임시로 로컬 파일을 읽는 중).
편집 내용이 기록될 대상 탭을 만들어야 합니다.

1. 「학생 졸업계획서」 구글 시트를 연다.
2. **파일 > 가져오기 > 업로드** → `C:\Users\1\Downloads\06_투고논문_트래커.xlsx` 선택.
3. 가져오기 위치: **"새 시트 삽입"** 선택 → 가져오기.
4. 새로 생긴 탭 이름이 **`06_투고논문`** 인지 확인(다르면 그 이름으로 변경).
   - 헤더(1행)에 `논문ID · 상태 · 상태변경일 · 투고(예정)일 · 게재(예정)일 · 제목 · 저널 …` 이 있으면 OK.
   - 스크립트가 열 위치를 **헤더 이름으로** 찾으므로 열 순서가 조금 달라도 동작합니다.

> 이 탭이 생기면 대시보드의 데이터 소스도 자동으로 로컬 파일 → 시트로 바뀝니다(파서가 탭을 우선).

## ② Apps Script 웹 앱 배포

1. 같은 시트에서 **확장 프로그램 > Apps Script**.
2. 기본 `Code.gs` 내용을 지우고, 이 폴더의 **`Code.gs` 전체**를 붙여넣기.
3. 맨 위 `SHARED_TOKEN` 값을 **길고 무작위한 문자열**로 교체(예: 영문·숫자 32자).
   - 이 값은 나중에 공개 사이트의 JS에도 들어갑니다. 계획서 §4가 수용한 개인용 위협 모델과 일치하지만, 원하면 나중에 PIN 게이트를 추가할 수 있습니다.
4. 💾 저장.
5. **배포 > 새 배포** → 톱니바퀴 > **웹 앱** 선택.
   - 설명: 아무거나 (예: `papers writer v1`)
   - **실행 계정: 나(your email)**
   - **액세스 권한: 링크가 있는 모든 사용자**
   - **배포** 클릭.
6. 권한 승인 창이 뜨면 본인 계정으로 **허용**(시트 편집 권한을 스크립트에 부여).
7. 완료 화면의 **웹 앱 URL** 복사 (`https://script.google.com/macros/s/AKfy…/exec`).

### 배포 확인(선택)
복사한 URL을 브라우저 주소창에 그대로 붙여 열면 `{"ok":true,"service":"sdclab papers writer","ping":true}` 가 보이면 정상입니다.

## ③ 저에게 전달할 것

- **웹 앱 URL** (`…/exec`)
- 설정한 **SHARED_TOKEN** 값

이 두 개를 받으면 제가:
1. `/papers` 칸반에 드래그 앤 드롭 연결(낙관적 반영 → Apps Script POST → 상태·상태변경일 시트 기록 → 성공 토스트 / 실패 시 원위치 롤백),
2. 연월·제목·저널 인라인 편집,
3. `/papers` 라이브 로드 전환(편집 즉시성·새로고침 일관성),
4. 드롭 → 시트 셀이 실제로 바뀌는지 왕복 테스트

를 진행합니다.

> 재배포가 필요할 때(코드 수정 후): **배포 > 배포 관리 > 편집(연필) > 버전: 새 버전 > 배포**. URL은 그대로 유지됩니다.
