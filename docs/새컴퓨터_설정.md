# 다른 컴퓨터에서 이어서 작업하기

이 프로젝트의 **모든 원본은 GitHub 저장소**에 있습니다 →
**https://github.com/jaeho19/sdclab-dashboard**

> 콘텐츠(과제·강의·연계 데이터)도 전부 저장소 안 `content/` 에 들어 있으므로,
> 클론만 하면 지금까지 작업한 내용이 그대로 이어집니다.
> `node_modules`, `dist`, `.astro`, `.netlify` 등은 자동 생성/로컬 전용이라 저장소에 없고, 아래에서 다시 만듭니다.

## 새 컴퓨터 1회 설정

### 0) 필요 프로그램
- **Node.js 22+** : https://nodejs.org
- **Git** : https://git-scm.com
- (선택) GitHub CLI `gh`

### 1) 저장소 클론
Obsidian에서 편집하려면 **볼트 안**에 클론하세요(이 컴퓨터와 같은 위치 권장):
```powershell
cd "<당신의 Obsidian 볼트 경로>"
git clone https://github.com/jaeho19/sdclab-dashboard.git "06_📋_연구대시보드"
cd "06_📋_연구대시보드"
```

### 2) 의존성 설치
```powershell
npm install
```

### 3) 깃 인증 (push 가능하게)
```powershell
gh auth login                 # 또는 git 자격 증명 설정
git config user.name  "jaeho19"
git config user.email "jaeho19@gmail.com"
```

### 4) Netlify 연결 (배포)
```powershell
npm i -g netlify-cli
netlify login                              # 브라우저 1회 로그인 (jaeho19@gmail.com)
netlify link --name sdclab-dashboard-156   # 기존 사이트에 연결
```
- 사이트 URL: https://sdclab-dashboard-156.netlify.app
- 사이트 ID: `c113708b-1106-46eb-bb94-4503818fa0aa`

### 5) 동작 확인
```powershell
npm run dev      # http://localhost:4321
```

### 6) 자동 배포 스케줄러 등록 (Windows)
PowerShell에서 (경로를 새 컴퓨터의 실제 경로로 바꿔서):
```powershell
$p = "<클론한 경로>\06_📋_연구대시보드\scripts\autosync.ps1"
$action  = New-ScheduledTaskAction -Execute 'powershell.exe' `
  -Argument "-NoProfile -ExecutionPolicy Bypass -WindowStyle Hidden -File `"$p`""
$trigger = New-ScheduledTaskTrigger -Once -At (Get-Date) `
  -RepetitionInterval (New-TimeSpan -Minutes 10)
Register-ScheduledTask -TaskName 'SDCLab-Dashboard-AutoSync' -Action $action -Trigger $trigger -Force
```

## 두 컴퓨터를 오갈 때 (중요)

- **작업을 시작하기 전에 항상 `git pull`** 하세요. 다른 컴퓨터의 autosync가 자동으로 푸시해 둔 최신 내용을 받아옵니다.
  ```powershell
  cd "...\06_📋_연구대시보드"; git pull
  ```
- 편집 → 저장하면 autosync(10분 주기) 또는 `scripts\autosync.ps1` 수동 실행이
  **커밋·푸시 + Netlify/GitHub Pages 배포**까지 처리합니다.
- autosync는 푸시 전에 `git pull --rebase`로 원격 변경을 먼저 합칩니다.
  같은 파일을 양쪽에서 동시에 고쳐 **충돌**이 나면 그 회차는 멈추고
  `원격과 충돌 — 수동으로 git pull 후 해결 필요` 메시지를 남깁니다 → 직접 `git pull`로 해결.

## Obsidian Sync를 함께 쓴다면 (권장 설정)
이 폴더는 **git으로 동기화**하는 것이 안전합니다. Obsidian Sync가 이 폴더까지 동기화하면
git과 충돌할 수 있으니, **Obsidian Sync 선택적 동기화에서 `06_📋_연구대시보드`(특히 `node_modules`)를 제외**하고
git에 맡기는 것을 권장합니다. (Obsidian은 동기화 여부와 상관없이 폴더 내용을 화면에 보여줍니다.)
