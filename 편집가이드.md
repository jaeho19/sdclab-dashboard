## 1. 어디를 편집하나요?

Obsidian에서 다음 폴더를 엽니다:

```
06_📋_연구대시보드/content/projects/
├─ forest-equity.md              ← 산림청 녹지형평성
```

## 2. 진행률·상태 바꾸기 (설정부)

```yaml
---
status: 진행중        # 초기 / 진행중 / 마무리 / 완료 중 하나
progress: 28          # 0~100 사이 숫자 (%)
---
```

- `progress` 숫자만 바꾸면 카드의 **진척 막대**가 즉시 반영됩니다.
- `status` 는 우측 상단 **배지** 색을 바꿉니다.

## 3. 일정(타임라인) 추가·수정하기

설정부의 `events:` 목록에 한 줄 추가하면 **통합 타임라인과 일정표**에 자동 반영됩니다.

```yaml
events:
  - { date: 2026-07-15, label: 빈집 데이터 교차검증, type: 마일스톤, status: 예정 }
```

| 항목 | 쓸 수 있는 값 | 설명 |
|------|--------------|------|
| `date` | `2026-07-15` 또는 `2026-08` | 날짜(일까지) 또는 월까지. 따옴표 없이 써도 됩니다. |
| `label` | 자유 텍스트 | 일정 이름 |
| `type` | 회의 / 마일스톤 / 제출 / 마감 / 보고 | 색·분류 |
| `status` | 완료 / 진행 / 예정 | 점 채움(완료=회색, 진행=주황, 예정=흰색) |

> 일정이 끝나면 `status` 를 `예정` → `완료` 로만 바꿔주면 됩니다.

## 3-A. 연구 흐름도(단계 다이어그램) 수정·갱신하기

각 과제 상세 페이지에는 **연구 흐름도**가 표시됩니다. **왼쪽**에는 단계가 위→아래로 이어지는 **다이어그램**(진행 스파인 + 노드 + 단계명), **오른쪽**에는 각 단계의 **세부 작업 리스트**가 나옵니다. 세부 작업은 클릭 없이 모두 펼쳐져 색상으로 중요도가 구분되고, 세로 진행선이 현재 단계까지 채워져 진행 위치를 한눈에 보여줍니다. 데이터는 설정부의 `flow:` 한 곳만 고치면 됩니다.

```yaml
flow:
  note: "이 연구의 전체 흐름을 2~4문장으로. 지금 어느 단계인지 명시."   # 다이어그램 하단 설명
  stages:
    - label: "가. 실태조사"          # 단계 이름
      status: 완료                  # 완료 / 진행 / 다음주 / 예정 (아래 표)
      summary: "한 줄 요약 (단계 카드에 표시)"
      period: "2026-03~04"          # 선택
      icon: "🔍"                    # 선택 — 생략하면 라벨로 자동 추정, 그래도 없으면 번호 표시
      steps:                        # 세부 항목 (토글 시 펼쳐짐)
        - { text: "세부 작업 1", status: 완료 }
        - { text: "세부 작업 2", status: 진행 }
        - { text: "세부 작업 3", status: 다음주 }
        - { text: "세부 작업 4", status: 예정 }
```

| `status` 값 | 색 | 뜻 |
|------------|----|----|
| `완료` | 초록 | 끝난 단계·작업 |
| `진행` | 파랑(테두리 강조) | **현재** 진행 중 (보통 단계 1개) |
| `다음주` | 주황 | **다음 주에 할 업무** (가장 최신 회의의 다음 할 일) |
| `예정` | 회색 점선 | 향후 계획 |

> **회의가 끝나면**: 끝난 작업의 `status` 를 `완료` 로, 다음 회의에서 하기로 한 일을 `다음주` 로, 한 단계가 마무리되면 다음 단계 `status` 를 `진행` 으로 바꿔주세요. 단계가 새로 늘면 `stages:` 에 한 블록을 추가하면 왼쪽 다이어그램과 오른쪽 세부 리스트가 자동으로 늘어납니다.

## 4. 회의/진행/할 일 내용 고치기 (본문)

`---` 아래 본문은 일반 마크다운입니다. `## 회의 내용`, `## 현재 진행상황`,
`## 앞으로 할 일` 제목 아래 글을 자유롭게 추가·수정하세요. 표·목록·굵게 모두 됩니다.

## 5. 과제를 새로 추가하려면

`content/projects/` 에 새 `.md` 파일을 하나 만들고(예: `new-project.md`),
기존 파일의 설정부를 복사해 값만 바꾸면 됩니다. `order:` 숫자가 카드 정렬 순서입니다.
파일을 추가하면 메인 화면에 카드와 타임라인 행이 자동으로 늘어납니다.

## 6. 저장하면 어떻게 배포되나요?

1. Obsidian에서 내용을 편집·저장한다.
2. **자동 동기화 스크립트**(`scripts/autosync.ps1`)가 변경분을 `main` 브랜치에 올리고(원본 보관),
   사이트를 빌드해 `gh-pages` 브랜치로 배포한다. *(아래 7번에서 1회 설정)*
3. **GitHub Pages**가 `gh-pages` 브랜치를 그대로 서빙한다 (반영까지 약 1분).
4. 주소에서 갱신된 내용 확인.

> 설정부 YAML에 오타(예: 따옴표·들여쓰기 깨짐)가 있으면 빌드가 실패해 그 회차 배포가 멈추지만,
> **직전 정상 버전이 그대로 유지**되므로 사이트가 깨지지 않습니다. 오타만 고쳐 다시 저장하세요.

> 빌드는 여러분 PC에서 일어나므로 `node_modules/` 가 있어야 합니다(이미 설치됨).
> 폴더를 옮겼다면 한 번 `npm install` 만 다시 해주세요.

## 7. 자동 동기화 1회 설정 (Windows 작업 스케줄러)

PowerShell에서 아래를 한 번 실행하면 **10분마다** 변경분을 자동 푸시합니다:

```powershell
$action  = New-ScheduledTaskAction -Execute 'powershell.exe' `
  -Argument '-NoProfile -WindowStyle Hidden -File "C:\Users\1\obsidian sync\06_📋_연구대시보드\scripts\autosync.ps1"'
$trigger = New-ScheduledTaskTrigger -Once -At (Get-Date) `
  -RepetitionInterval (New-TimeSpan -Minutes 10)
Register-ScheduledTask -TaskName 'SDCLab-Dashboard-AutoSync' `
  -Action $action -Trigger $trigger -Description '대시보드 콘텐츠 자동 배포'
```

- **수동으로 한 번에 올리고 싶을 때:** `scripts/autosync.ps1` 더블 실행, 또는
  ```powershell
  cd "C:\Users\1\obsidian sync\06_📋_연구대시보드"; .\scripts\autosync.ps1
  ```
- **자동 동기화 끄기:** `Unregister-ScheduledTask -TaskName 'SDCLab-Dashboard-AutoSync'`

## 8. Netlify로 전환하기 (선택)

지금은 GitHub Pages로 배포됩니다. Netlify로 바꾸려면:

1. [app.netlify.com](https://app.netlify.com) 로그인 → **Add new site → Import an existing project** → GitHub `sdclab-dashboard` 저장소 선택.
2. 빌드 설정은 `netlify.toml`에 이미 있습니다 (`build: npm run build`, `publish: dist`). 그대로 Deploy.
3. 끝. 이후 push마다 Netlify가 자동 배포합니다. (이때는 사이트 주소가 루트라 base 경로 설정도 자동 처리됩니다.)

> GitHub Pages와 Netlify를 **동시에** 써도 됩니다(같은 저장소, 두 주소).
